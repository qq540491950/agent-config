# Model Route

检查当前模型路由配置（如存在），并输出摘要与建议。

## 命令功能

- 展示当前路由规则与模型选择策略
- 标出潜在冲突或不一致
- 给出改进建议（如有）

## 执行步骤

1. 读取所有 `agents/*.md` 文件的 `model:` frontmatter 字段
2. 输出当前路由表：agent 名称 → model 映射
3. 标出使用 `opus` 的 agent（添加「高成本」提示）
4. 统计各模型使用数量（sonnet / opus / haiku）
5. 建议：高频低复杂度任务是否可降级到 haiku 以节省成本

## 何时使用

- 需要核对模型策略是否生效时
- 迁移模型或成本优化前
