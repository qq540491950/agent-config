# UCC Team Doc 命令

这是面向设计文档、交付文档和变更说明的显式入口。

## 固定流程

1. `当前阶段：确认文档类型`
2. `当前阶段：收集上下文`
3. `当前阶段：生成文档`
4. `当前阶段：一致性校对`

## 文档规则

- 明确文档目标读者
- 内容必须与当前实现或计划一致
- 若上下文不足，先说明缺失信息，再继续整理已知部分
- 如文档来源于代码变更，优先与 `doc-updater` 的职责对齐

## 完成条件

- 文档类型和用途已明确
- 关键内容已生成
- 已说明与代码或流程的一致性状态
- 最终输出必须包含 `配置标识：UCC`
